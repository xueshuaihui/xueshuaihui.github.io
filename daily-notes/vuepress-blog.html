# 搭建 vuepress 项目

根据官方文档先把一个 vuepress 项目拉起来，vuepress 官网 (opens new window)

  1. # 创建并进入一个新目录

    mkdir vuepress-starter && cd vuepress-starter
    1

  2. # 使用你喜欢的包管理器进行初始化

    yarn init # npm init
    1

  3. # 将 VuePress 安装为本地依赖

    yarn add -D vuepress # npm install -D vuepress
    1

  4. # 创建你的第一篇文档

    mkdir docs && echo '# Hello VuePress' > docs/README.md
    1

  5. # 在 package.json 中添加快捷执行脚本

    {
	"scripts": {
		"docs:dev": "vuepress dev docs",
		"docs:build": "vuepress build docs"
	}
}
    1
    2
    3
    4
    5
    6

  6. # 在本地启动服务器

    yarn docs:dev # npm run docs:dev
    1

# 部署 GitHub page

  1. # github 配置

    代码仓库与 GitHub page 部署仓库分开 好处就是代码更改不会影响到线上发布的页面

    代码仓库如何创建、提交就不说了，直接说 GitHub page 相关的操作配置

    1. 登录 GitHub 之后，在页面右上角点击 + 加号按钮，点击 New repository。 仓库的名称需要安装 GitHub page 规定来写。
      规则就是：

      YOUR-GITHUB-USERNAME.github.io
      1

    2. 创建成功之后，进入了项目主页面。点击设置按钮 setting -> pages 查看配置

  2. # 项目配置

    1. 根据 vuepress 官网创建 deploy.sh 脚本
      vuepress#github-pages (opens new window)

      #!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'
# git branch -M master
# git remote add master https://github.com/YOUR-GITHUB-USERNAME.io.git
// YOUR-GITHUB-USERNAME 需要改成你自己的名称
# git push -u master master
# 如果发布到 https://<USERNAME>.github.io
git push -f https://github.com/YOUR-GITHUB-USERNAME.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

    2. 自动部署

      这里没有按照官网提示进行自动部署
      而是采用了 pre-push 插件，在提交代码时进行 sh 脚本触发实现自动部署
      具体步骤如下

      1. 项目安装 pre-push
        npm install --save-dev pre-push
        1
      2. 配置
        package.json 中添加
      {
    "scripts": {
        "deploy": "bash deploy.sh" // 执行sh脚本
    },
    "pre-push": [
        "deploy" // 提交代码前之前npm run deploy
    ]
}
